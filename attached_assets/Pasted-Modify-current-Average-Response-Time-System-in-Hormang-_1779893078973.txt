Calculate and continuously update each user’s average response time from real conversations.

providers

The metric should reflect:

How quickly this user typically replies to incoming messages
Core calculation logic

Average response time is calculated from:

time between receiving a message
→ and sending the first reply back

Provider

IMPORTANT — Response pairing rules

This part matters.

Measure only:

first reply to latest incoming message
Valid

Customer:

hello

Provider:

salom

→ count interval

Invalid duplicate counting

Customer sends:

hello
are you available?
please answer

Provider sends one reply.

This should count as:

one response interval only

Not three.

Recommended implementation

For each chat:

Track pending unanswered incoming message per participant.

Pseudo:

lastIncomingAwaitingReply[userId]

When user replies:

compute:

replyTimestamp - incomingTimestamp

Store metric.

Then clear pending.

Data model

Add to user profile:

averageResponseTimeMinutes?: number
responseSamplesCount?: number
responseSamples?: number[]

Preferred:

Store running values:

responseTimeTotalMinutes
responseSamplesCount

Then derive average.

More efficient than storing every sample forever.

Example
provider.responseTimeTotalMinutes += 20

provider.responseSamplesCount += 1

provider.averageResponseTimeMinutes =
total / count
Update trigger

Recalculate whenever:

new message sent

Specifically:

when message is a reply to unanswered incoming message.
