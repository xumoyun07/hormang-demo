# Implement Complete "Xizmatlar Tarixi" (Service History) Feature for Providers

You are working on Hormang, a marketplace platform similar to Soomgo where customers create service requests and providers send offers.

Implement a complete Provider Service History system.

## Goal

Providers should be able to see all completed services, their earnings, ratings, customer reviews, statistics, and portfolio-worthy projects.

This feature should feel professional and production-ready.

---

# 1. Create Service History Data Model

Create a ServiceHistory entity.

Fields:

```ts
interface ServiceHistory {
  id: string;

  providerId: string;
  customerId: string;

  requestId: string;
  offerId: string;

  categoryId: string;
  categoryName: string;

  serviceTitle: string;
  serviceDescription: string;

  finalPrice: number;

  status: "completed";

  rating?: number;
  review?: string;

  completedAt: Timestamp;

  durationMinutes?: number;

  beforePhotos?: string[];
  afterPhotos?: string[];

  locationName?: string;

  isRepeatCustomer: boolean;

  isPortfolio: boolean;
}
```

Important:

Store a snapshot when service is completed.

Do NOT rely on original request data later.

---

# 2. Automatically Create History Record

When:

Request.status === "completed"

Create ServiceHistory record automatically.

Copy:

* service title
* category
* final price
* customer
* provider
* completion date
* review/rating

This record must remain immutable.
