You are working on a React + TypeScript + Tailwind + Supabase (or localStorage fallback) service marketplace project in Replit.

## 🎯 Goal

Implement a **full Audit Log System** that tracks meaningful system actions, stores them, and displays them in an admin UI.

This system must include:

* storage layer (Supabase or localStorage fallback)
* reusable logging utility (hooks/functions)
* automatic tracking for key actions
* admin UI with filters and details

---

# 🧠 Core Principle

Log only **high-value, traceable actions**:

* Admin actions
* Marketplace lifecycle events
* Financial (Tanga) changes
* Referral events
* Risk/flag actions

DO NOT log UI interactions (clicks, scrolls, etc.)

---

# ✅ 1. Data Model

Create AuditLog type:

```ts id="audit_type"
type AuditLog = {
  id: string;
  actorId: string;
  actorRole: "admin" | "provider" | "customer" | "system";
  action: string;
  targetId?: string;
  targetType?: "user" | "request" | "offer" | "referral" | "tanga";
  description: string;
  metadata?: Record<string, any>;
  createdAt: string;
};
```

# ✅ 2. Storage Layer

Create table:

```sql id="audit_sql"
audit_logs (

  actor_id text,
  actor_role text,
  action text,
  target_id text,
  target_type text,
  description text,
  metadata jsonb,
  created_at timestamp
)
```

# ✅ 3. Logging Utility (CRITICAL)

Create reusable function:

```ts id="log_fn"
function logAction(log: Omit<AuditLog, "id" | "createdAt">) {
  const newLog: AuditLog = {
    ...log,
    id: crypto.randomUUID(),
    createdAt: new Date().toISOString(),
  };

  // Save to Supabase OR localStorage
}
```

---

## Create hook:

```ts id="hook"
export function useAuditLog() {
  return { logAction };
}
```
