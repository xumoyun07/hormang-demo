# Enhance Service Completion + Portfolio System

We already have:

* Service History
* Add to Portfolio toggle
* ServiceHistory model
* Provider Public Profile (PPP)

Now implement a professional project portfolio system based on REAL completed services.

## Goal

Portfolio projects must originate from completed services.

Providers should NOT create portfolio entries from scratch.

Portfolio should be generated from verified service history.

---

# 1. Add Completion Photos Workflow

Modify service completion flow.

When provider clicks:

"Complete Service"

Open completion modal.

Fields:

### Completion Photos

Upload After Photos

```ts
afterPhotos: string[]
```

Limit:

* max 10 photos
* jpg/png/webp

Optional but strongly encouraged.

UI:

Completed Work Photos

[ + Add Photos ]

Photo grid preview.

---

### Completion Notes

```ts
completionNotes?: string
```

Example:

"Deep cleaned kitchen and removed grease buildup."

Max 500 characters.

---

### Duration

Optional.

```ts
durationMinutes?: number
```

Examples:

120 minutes

3 days

Store numeric value.

---

### Confirm Completion

Save all data into ServiceHistory snapshot.

---

# 3. Create Portfolio Project Model

```ts
interface PortfolioProject {
  title: string;

  description: string;

  coverPhoto: string;

  additionalPhotos: string[];

  featured: boolean;

  createdAt: Timestamp;
}
```

Portfolio data belongs to ServiceHistory.

Do NOT create separate project entities yet.

# 5. Portfolio Creation Modal

Prepopulate data from completed service.

Fields:

### Project Title

Required.

Example:

Kitchen Deep Cleaning

Bathroom Renovation

English Tutoring Program

---

### Project Description

Required.

Max 1000 chars.

Example:

Completed deep cleaning of a 25m² kitchen including oven, cabinets and tile surfaces.

---

### Cover Photo

Choose from existing afterPhotos.

Provider must select one.

If no photos exist:

Show message:

"Upload completion photos first."

Disable portfolio creation.

---

### Additional Photos

Allow selecting from completion photos.

No re-upload needed.

---

### Featured Project

Toggle:

📌 Pin to Top

```ts
featured: boolean
```

Only 3 featured projects maximum.

---

### Save

Creates:

```ts
portfolioData
```

inside ServiceHistory.

Sets:

```ts
isPortfolio = true
```

# 10. PPP Performance

Load portfolio separately from history.

Only fetch:

```ts
isPortfolio === true
```

Lazy load photos.

Support future pagination.

---

# 11. Security

Portfolio is public.

Service history remains private.

Customer data must never appear in portfolio.

Remove all sensitive information before rendering.
