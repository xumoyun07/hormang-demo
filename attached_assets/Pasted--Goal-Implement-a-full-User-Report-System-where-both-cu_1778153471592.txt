
## 🎯 Goal

Implement a full **User Report System** where both customers and providers can report inappropriate users directly from the **Public Profile Preview Modal (PPP Modal)**.

The system must include:

* report UI inside PPP modal
* reporting modal/drawer
* backend storage logic
* admin moderation page
* safeguards against abuse
* clean, professional UX

---

# 🧠 Core UX Principle

Reporting should feel:

* contextual
* subtle
* serious
* easy to access

DO NOT make it visually aggressive.

---

## Location:

Public Profile Preview Modal (everywhere except in profile settings page)

---

* rounded-xl
* subtle hover states
* neutral styling

# ✅ 7. Backend Data Structure

Create new entity/table:

```text id="rg3a1m"
user_reports
```

DO NOT merge with generic feedback system.

---

## Suggested schema:

```ts id="9dhnjj"
{
  id: string,
  reporterUserId: string,
  reportedUserId: string,

  reason:
    | "spam"
    | "fake_profile"
    | "abuse"
    | "fraud"
    | "inappropriate_content"
    | "outside_contact"
    | "other",

  description?: string,

  attachments?: string[],

  status:
    | "new"
    | "in_review"
    | "resolved"
    | "dismissed",

  adminNote?: string,

  createdAt: string
}
```

# ✅ 8. Safeguards (VERY IMPORTANT)

## Prevent self-reporting

```ts id="5b9s4j"
if (reporterUserId === reportedUserId)
```

Block submission.

---

## Prevent spam reports

If same user reports same target within 24h:

* prevent duplicate report

---

## Require authentication

Only logged-in users can report.

# 🎨 13. UI Style Rules

Use:

* neutral colors
* subtle red/amber accents
* compact modern cards
* rounded-2xl
* mobile-first

# ⚠️ 14. DO NOT

* Do NOT expose report counts publicly
* Do NOT notify reported users
* Do NOT allow anonymous reports
* Do NOT use raw IDs in admin UI
