You are working on a React + TypeScript + Tailwind + Supabase (or localStorage fallback) service marketplace project in Replit.

## 🎯 Goal

Build an **Advanced Users Management System** inside `/admin/users` that acts as a **control center for customers and providers**, not just a list.

This system must support:

* user monitoring
* performance tracking
* trust/risk management
* admin actions
* referral visibility

---

# 🧠 Core Concept

Users = ALL participants:

* Customers
* Providers
* Dual-role users

Use ONE unified system with role-based data.

---

# ✅ 1. Main Users Table

Route:

```text
/admin/users
```

Create a powerful table with columns:

### Identity

* Name
* Phone
* Role (Customer / Provider / Both)

### Activity

* Last active
* Created date

### Performance (Provider only)

* Rating (⭐)
* Jobs completed
* Response rate (if available)

### Marketplace Value

* Total earned (provider)
* Total spent (customer)

### Referral

* Referral code
* Referred by
* Total referrals
* Tanga earned from referrals

### Trust / Risk

* Verification status (verified / not)
* Flags count
* Cancellation rate
